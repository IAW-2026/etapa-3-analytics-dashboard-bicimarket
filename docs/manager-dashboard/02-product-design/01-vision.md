# 2.1 — Product Vision

> **Manager Dashboard — Product Design**
>
> The vision, mission, and strategic goals for the BiciMarket Manager Dashboard.

---

## Vision Statement

A single pane of glass that gives BiciMarket leadership real-time visibility into the entire marketplace — revenue, operations, finance, sellers, customers, and products — with AI-powered insights that help them make better decisions faster.

## Mission

Aggregate data from all four BiciMarket applications (Buyer, Seller, Shipping, Payments) into a unified, role-appropriate dashboard that enables:

1. **Monitor**: Track KPIs across the marketplace in real-time
2. **Analyze**: Drill down into metrics to understand trends and anomalies
3. **Act**: Make informed decisions based on cross-system data
4. **Forecast**: Predict future performance using historical patterns
5. **Report**: Generate insights for stakeholders automatically

## Strategic Goals

| Goal | Metric | Timeframe |
|------|--------|-----------|
| Reduce time-to-insight for executives | From ad-hoc queries across 4 apps to < 30s in dashboard | MVP |
| Enable data-driven seller management | Seller performance metrics available in one view | V2 |
| Automate weekly reporting | 100% of weekly reports generated automatically | V3 |
| Predictive marketplace management | Revenue forecasts with < 15% error margin | V3 |
| Proactive issue detection | 90% of anomalies flagged before anyone notices | V2 |

## Core Value Proposition

**Before**: Managers log into 4 separate admin panels, export data to spreadsheets, manually join and analyze.

**After**: One dashboard shows everything — revenue trends, settlement status, seller performance, delivery bottlenecks — with AI answering questions in natural language.

## Success Criteria

- Managers use the dashboard as their primary daily tool (not the 4 individual admin panels)
- Time spent on manual reporting reduced by 80%
- Anomalies detected within 15 minutes of occurrence
- All P0 KPIs available within the first month of MVP launch

## Non-Goals (MVP)

- No data mutations (read-only dashboard)
- No customer-facing features
- No real-time event streaming (polling is sufficient)
- No multi-language support (Spanish/English deferred)
- No mobile app (responsive web only)
